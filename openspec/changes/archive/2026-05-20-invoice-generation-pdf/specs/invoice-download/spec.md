## ADDED Requirements

### Requirement: Invoice data model
The system SHALL define a `Factura` entity with a 1:1 relationship to `Pedido`, containing: `numero_factura` (autogenerated sequential), `fecha_emision` (UTC timestamp of generation), `cuit_cliente` (optional string, max 20 chars), `tipo_factura` (enum: A, B, C), and `monto_total` (float >= 0).

#### Scenario: Factura created with valid pedido
- **WHEN** a Factura is generated for an existing Pedido
- **THEN** the Factura SHALL have `numero_factura` set to the next sequential value
- **AND** `fecha_emision` SHALL be set to the current UTC timestamp
- **AND** `monto_total` SHALL match the Pedido's `total` field

#### Scenario: Factura cannot be duplicated
- **WHEN** a Factura already exists for a Pedido
- **THEN** the system SHALL reject the generation request with HTTP 409 Conflict

### Requirement: PDF generation with commercial invoice format
The system SHALL generate a PDF document using ReportLab containing: invoice header with `numero_factura` and `fecha_emision`, seller/buyer info, line-item table with product name snapshot, unit price snapshot, quantity, and subtotal per item, total amount, and footer with `tipo_factura` and `cuit_cliente` if provided.

#### Scenario: PDF contains all pedido details
- **WHEN** a Factura PDF is generated for a Pedido with 3 items
- **THEN** the PDF SHALL contain a table with exactly 3 rows, one per PedidoDetalle
- **AND** each row SHALL show `producto_nombre_snapshot`, `precio_unitario_snapshot`, `cantidad`, and `subtotal`

#### Scenario: PDF renders optional CUIT
- **WHEN** the Factura has a non-null `cuit_cliente`
- **THEN** the PDF SHALL display the CUIT in the buyer info section

#### Scenario: PDF renders without CUIT
- **WHEN** the Factura has a null `cuit_cliente`
- **THEN** the PDF SHALL omit the CUIT line and show "Consumidor Final" instead

### Requirement: Generate invoice endpoint
The system SHALL provide `POST /api/v1/pedidos/{pedido_id}/factura` that atomically: changes the Pedido estado to `FACTURADO` (if currently `ENTREGADO`), inserts a `Factura` record, and records the audit trail in `HistorialEstadoPedido`. Returns the created `Factura` as JSON.

#### Scenario: Successful invoice generation
- **WHEN** a POST is made to `/api/v1/pedidos/{id}/factura` for a Pedido in `ENTREGADO` state
- **THEN** the response SHALL be HTTP 201 with a JSON body containing the new Factura
- **AND** the Pedido estado SHALL be `FACTURADO`
- **AND** a new `HistorialEstadoPedido` row SHALL be created with estado `FACTURADO`

#### Scenario: Invoice rejected for non-terminal state
- **WHEN** a POST is made to `/api/v1/pedidos/{id}/factura` for a Pedido NOT in `ENTREGADO` state
- **THEN** the response SHALL be HTTP 400 with a descriptive error message

#### Scenario: Invoice rejected for duplicate generation
- **WHEN** a POST is made to `/api/v1/pedidos/{id}/factura` and a Factura already exists for that Pedido
- **THEN** the response SHALL be HTTP 409 Conflict

### Requirement: Download invoice PDF endpoint
The system SHALL provide `GET /api/v1/pedidos/{pedido_id}/factura` that returns the generated PDF as a `StreamingResponse` with content-type `application/pdf` and content-disposition `attachment; filename="factura-{numero_factura}.pdf"`.

#### Scenario: Successful PDF download
- **WHEN** a GET is made to `/api/v1/pedidos/{id}/factura` for a Pedido that has a Factura
- **THEN** the response SHALL have status 200
- **AND** content-type SHALL be `application/pdf`
- **AND** content-disposition SHALL contain `attachment; filename="factura-{numero}.pdf"`

#### Scenario: 404 when no invoice exists
- **WHEN** a GET is made to `/api/v1/pedidos/{id}/factura` for a Pedido without a Factura
- **THEN** the response SHALL be HTTP 404

### Requirement: Frontend download button (cliente)
The system SHALL show a "Descargar Factura" button on `PagoResultPage` when the payment was successful and the Pedido has a Factura available. Clicking SHALL trigger a PDF download via the browser's native download mechanism.

#### Scenario: Download button visible after invoice generation
- **WHEN** a Factura exists for a successfully paid Pedido
- **THEN** the `PagoResultPage` SHALL display a "Descargar Factura" button
- **AND** clicking it SHALL download the PDF file

### Requirement: Frontend download button (admin)
The system SHALL show a "Descargar Factura" button in the admin pedidos list/toggle for any Pedido that has an associated Factura.

#### Scenario: Admin can download any existing invoice
- **WHEN** an admin views the pedidos list and a Pedido has a Factura
- **THEN** a "Descargar Factura" button SHALL be displayed for that row
- **AND** clicking it SHALL download the PDF file
